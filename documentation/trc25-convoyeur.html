Convoyeur 2.0

1. Vers convoyeur 2.0

Lors de la phase de présélection de la TRC 2025, une première version du convoyeur destiné au tri et au transport des déchets a été conçue et testée. Bien que fonctionnelle, cette version initiale présentait plusieurs limites, tant sur le plan mécanique qu’électronique.
La conception du Convoyeur 2.0 vise à corriger ces limites en se concentrant sur deux axes principaux :

  • Améliorations mécaniques: remplacement du bois par des pièces imprimées en 3D, structure démontable en quatre parties, système d'ajustement de tension du tapis optimisé et supports de capteurs repensés.
  • Améliorations électroniques: meilleure intégration des capteurs, câblage optimisé, support moteur stabilisé et fiabilisation générale du fonctionnement.

L’objectif du Convoyeur 2.0 est d’offrir une structure plus solide, modulable, facilement réparable et mieux adaptée à l’intégration dans le système de tri intelligent pour la TRC 2025.

2. Améliorations mécaniques

Le Convoyeur 2.0 intègre plusieurs améliorations mécaniques visant à rendre le système plus robuste, modulable et facile à entretenir, tout en conservant les éléments fonctionnels essentiels de la version précédente (roulements, tambours, moteur, ...).

a. Structure démontable

La structure initiale en bois a été remplacée par des pièces imprimées en 3D. Le convoyeur est désormais entièrement démontable, découpé en quatre parties principales, ce qui facilite le transport, l’assemblage et la maintenance.

b. Système d’ajustement de la tension du tapis

Un nouveau mécanisme permet de régler précisément la tension du tapis. Ce système est composé de deux vis filetées et de trois écrous montés autour d’un support central accueillant le tambour arrière. En vissant ou dévissant, les écrous se déplacent axialement, entraînant le support et ajustant la tension du tapis pour éviter tout glissement.


c. Supports

  • Support du capteur de fin de convoyage: intégré et conçu pour maintenir le capteur en position stable et fiable à la sortie du convoyeur.
  • Support du capteur ultrason: conçu pour maintenir le capteur en position stable et précise afin de détecter la présence des déchets sur le convoyeur.
  • Supports de l'alimentation: pour fixer et sécuriser le système d’alimentation.
  • Supports du pupitre et des circuits: pour les boutons de contrôle et la LED de signalisation, ainsi que les circuits.
  • Supports du tapis: pour soutenir et maintenir le tapis en place.

d. Autres pièces supplementaires

  • Pièce de fixation du moteur: conçue pour assurer le maintien, l’alignement correct et la stabilité du moteur.
  • Pièce de fin de convoyage: placée à l’extrémité du convoyeur afin de guider et recentrer les déchets vers le milieu du tapis avant la phase de détection et de tri.

e. Assemblage sous SolidWorks

Toutes les pièces ont été modélisées sur SolidWorks, puis assemblées sous SolidWorks.

f. Impression et assemblage

Après modélisation sur SolidWorks et validation des pièces par l’équipe mécanique, toutes les pièces ont été imprimées en 3D puis assemblées pour constituer le convoyeur.


3. Améliorations électroniques

Le Convoyeur 2.0 intègre plusieurs améliorations électroniques pour renforcer la fiabilité, la flexibilité et la connectivité du système.

a. Alimentation

Le Convoyeur 2.0 conserve l’alimentation par batterie de la version précédente, garantissant ainsi la portabilité du système et son fonctionnement même en l’absence d’une source secteur. Cette configuration permet de déplacer facilement le convoyeur et de l’utiliser dans différents environnements sans dépendre d’une prise électrique.
Pour accroître la flexibilité et permettre un fonctionnement prolongé, un système d’alimentation par prise secteur a été ajouté. Cette option fournit une source d’énergie alternative et continue, réduisant la dépendance aux batteries et assurant une disponibilité permanente lors de tests prolongés ou d’une utilisation intensive.

b. Communication et connectivité

  • Communication I2C: une interface I2C a été ajoutée pour permettre une liaison rapide et fiable entre le convoyeur et le DOFBOT. Cette communication facilite l’échange de données et le contrôle des différents modules du système, tout en simplifiant le câblage et l’intégration.
  • Module Wi-Fi ESP01: le module Wi-Fi ESP01 a été intégré afin de transmettre les données du convoyeur en temps réel. Il permet la supervision à distance du fonctionnement, le suivi des capteurs et le contrôle du convoyeur. Cette connectivité améliore la flexibilité du système et facilite le diagnostic et la maintenance.

Pour assurer la compatibilité des deux communications, un régulateur de tension 5V à 3,3V a été utilisé afin d'alimenter correctement les modules nécessitant une tension plus faible. Un convertisseur de niveau logique a également été intégré pour adapter les signaux entre le microcontrôleur et les périphériques, garantissant ainsi une communication fiable et sécurisée pour l'I²C et le module Wi-Fi ESP01.

Convertisseur de niveau logique bidirectionnel 3.3V-5V

c. Commandes

  • Marche/Arrêt: Les deux boutons poussoirs ont été remplacés par un interrupteur à bascule, offrant un contrôle plus pratique et moins encombrant.
  • Reset du convoyeur un bouton de réinitialisation a été ajouté pour permettre de redémarrer rapidement le microcontrôleur en cas de besoin, facilitant la maintenance et le débogage.

d. Câblage

Le câblage a été réorganisé afin de faciliter la maintenance et de réduire les risques de court-circuit. Toutes les connexions sont sécurisées et facilement accessibles, garantissant ainsi un fonctionnement stable et fiable du convoyeur.

4. Programmation

    Le microcontrôleur du convoyeur a été programmé pour :
  • Lire en temps réel les capteurs(ultrason et infrarouge).
  • Contrôler le moteur.
  • Assurer la communication avec le DOFBOT.
  • Réagir aux commandes utilisateur(interrupteur à bascule, bouton reset) et sécuriser le fonctionnement du système.

Le code est structuré de manière modulaire, facilitant la maintenance et les futures améliorations.

code_convoyeur.ino 

#include <SoftwareSerial.h>

#define pinStart 2

// Contrôle moteur
#define pinMotor 3

// Définir les broches du capteur de couleur
#define LED_R 9
#define LED_G 10
#define LED_B 11

// capteur infrarouge
#define infraredSensor 8

// capteur Ultrason
#define trigPin 6
#define echoPin 5

// RX, TX
SoftwareSerial espSerial(4, 7);

unsigned long distance = 0, lastInterrupt = 0, lastSend = 0;
bool finTapis = true;
volatile bool is_on = false;

const int vitesse = 250;



void setup()
{
    // Moteur
    pinMode(pinMotor, OUTPUT);
    analogWrite(pinMotor, 0); // arrêt initial

    // LED
    pinMode(LED_R, OUTPUT);
    pinMode(LED_G, OUTPUT);
    pinMode(LED_B, OUTPUT);
    setColor(0, 0, 0); // blanc

    // Ultrason
    pinMode(trigPin, OUTPUT);
    pinMode(echoPin, INPUT);

    // Infrarouge
    pinMode(infraredSensor, INPUT);

    // interrupteur à bascule
    pinMode(pinStart, INPUT);
    attachInterrupt(digitalPinToInterrupt(pinStart), interrupt_routine, CHANGE);

    Serial.begin(9600);
    espSerial.begin(9600); 
    delay(1000);
    is_on = digitalRead(pinStart);
}


void loop()
{
  if (!is_on)
  {
    // led en rouge pour indiquer que le système est en stop
    setColor(255, 0, 0);
    analogWrite(pinMotor, 0);
    while (!is_on)
    {      
      delay(100);
    }
  }
  else
  {
    distance = getDistance();
    int infraredValue = digitalRead(infraredSensor);

    if (infraredValue)
    {
      analogWrite(pinMotor, 0);
      finTapis = true;
      setColor(255, 165, 0);
      send_info();
    }

    else if (distance < 50 || !finTapis)
    {
      setColor(0, 255, 0);
      if(distance < 15){
        delay(500);
        finTapis = false;
        lastSend = 0;
        analogWrite(pinMotor, vitesse);
      }
    else if (distance > 45)
    {
      analogWrite(pinMotor, vitesse-50);
      }
    }
    else
    {
      setColor(0, 0, 255);
      analogWrite(pinMotor, 0);
    }
  }
}

// Fonction pour lire la distance avec le capteur ultrason
long getDistance()
{
    digitalWrite(trigPin, LOW);
    delayMicroseconds(2);
    digitalWrite(trigPin, HIGH);
    delayMicroseconds(10);
    digitalWrite(trigPin, LOW);

    long duration = pulseIn(echoPin, HIGH);
    long distance = (duration / 2.0) * 0.0344;
    return distance;
}

void setColor(unsigned int red, unsigned int green, unsigned int blue)
{
    red = map(red, 0, 255, 255, 0);
    green = map(green, 0, 255, 255, 0);
    blue = map(blue, 0, 255, 255, 0);
    analogWrite(LED_R, red);
    analogWrite(LED_G, green);
    analogWrite(LED_B, blue);
}

void send_info(){
  unsigned long currentMillisInfo = millis();
  if (currentMillisInfo - lastSend > 2000) {
    espSerial.write("piece");
    Serial.println("piece");
    lastSend = currentMillisInfo;
  }
}

   
void interrupt_routine() {
    unsigned long currentMillis = millis();
    detachInterrupt(digitalPinToInterrupt(pinStart)); 
    if (currentMillis - lastInterrupt > 200) { // anti-rebond 100ms
      is_on = digitalRead(pinStart);
      lastInterrupt = currentMillis;
    }
   attachInterrupt(digitalPinToInterrupt(pinStart), interrupt_routine, CHANGE);
}

Test de la pupitre de commande

test_pupitre.ino 

#define pinStart 2
#define LED_R 9
#define LED_G 10
#define LED_B 11

unsigned long lastInterrupt = 0;
volatile bool is_on = false;

// variables pour gestion du timing
unsigned long previousMillis = 0;
int colorState = 0;
const unsigned long interval = 2000; // intervalle de 2 secondes

void setup()
{
  Serial.begin(9600);
    pinMode(LED_R, OUTPUT);
    pinMode(LED_G, OUTPUT);
    pinMode(LED_B, OUTPUT);
    setColor(0, 0, 0); // éteint la LED

    pinMode(pinStart, INPUT);
    attachInterrupt(digitalPinToInterrupt(pinStart), interrupt_routine, CHANGE);
    delay(500);
    is_on = digitalRead(pinStart);
    Serial.println(is_on);
}

void loop() {
    unsigned long currentMillis = millis();

    if (is_on) {
        if (currentMillis - previousMillis >= interval) {
            previousMillis = currentMillis;

            // changer la couleur en fonction de colorState
            switch(colorState) {
                case 0:
                    setColor(250, 0, 0); // rouge
                    break;
                case 1:
                    setColor(0, 250, 0); // vert
                    break;
                case 2:
                    setColor(0, 0, 250); // bleu
                    break;
            }

            colorState++;
            if (colorState > 2) colorState = 0;
        }
    } else {
        setColor(0, 0, 0); // LED éteinte
        previousMillis = currentMillis; // réinitialiser le timing
        colorState = 0;
    }
}

void setColor(unsigned int red, unsigned int green, unsigned int blue)
{
    red = map(red, 0, 255, 255, 0);
    green = map(green, 0, 255, 255, 0);
    blue = map(blue, 0, 255, 255, 0);
    analogWrite(LED_R, red);
    analogWrite(LED_G, green);
    analogWrite(LED_B, blue);
}

void interrupt_routine() {
    unsigned long currentMillis = millis();
    detachInterrupt(digitalPinToInterrupt(pinStart)); 
    if (currentMillis - lastInterrupt > 200) { // anti-rebond 100ms
      is_on = digitalRead(pinStart);
      Serial.println(is_on);
      lastInterrupt = currentMillis;
    }
  attachInterrupt(digitalPinToInterrupt(pinStart), interrupt_routine, CHANGE);
}
