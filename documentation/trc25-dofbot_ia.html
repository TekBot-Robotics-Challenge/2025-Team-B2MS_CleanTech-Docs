Détection et tri - Dofbot

Cette section est consacrée à la détection et au tri automatique des déchets à l’aide du bras robotisé Dofbot, élément central du système intelligent de tri développé dans le cadre du projet TRC25. Après l’étude du modèle de détection basé sur YOLO, l’objectif est désormais d'implementer le modèle et d’assurer une prise de décision autonome.

    Cette section détaille :
  • l’intégration du modèle YOLO dans le Dofbot,
  • le processus de détection et de validation des déchets,
  • la logique de tri adoptée,
  • ainsi que la commande et la synchronisation des mouvements du bras robotisé.

1. Détection des dechets

a. Choix du modèle et contraintes d’intégration

Lors de la section Modèle de détection YOLO, une étude comparative a été menée entre YOLOv5 et YOLOv8. Les résultats obtenus ont montré des performances similaires.

Cependant, ces deux modèles, entraînés avec la bibliothèque Ultralytics, ont posé des problèmes de compatibilité lors de leur intégration dans le Dofbot. En effet, le Dofbot utilise Python 3.6.9 et dispose de versions antérieures de plusieurs bibliothèques telles que numpy, matplotlib ou torch, ce qui rend la mise à jour complexe et susceptible d’introduire d’autres incompatibilités.

Afin de garantir une intégration stable, maîtrisée et exploitable, il a été nécessaire de réentraîner le modèle de détection en utilisant YOLOv5 basé sur PyTorch.

b. Entrainement du modèle YOLOv5 Pytorch

L’entraînement du modèle YOLOv5-PyTorch pour le Dofbot suit globalement la même démarche que celle utilisée avec Ultralytics, avec quelques points spécifiques à prendre en compte pour garantir la compatibilité.

  • Téléchargement des images labellisées : Il est essentiel de s’assurer que les annotations et les images sont compatibles avec YOLOv5-PyTorch et non avec la version Ultralytics utilisée précédemment. Il faut alors utiliser la version YOLOv5-Pytorch.
  • La version utilisée: Les modèles de detection YOLO ne cesse d'être mise à jour, il faut donc faire attention à la version utilisée.
  • les fichiers yolov5.yaml: nc : 3 #nombre de classes
    Ces fichiers décrivent la structure du réseau de neurones YOLOv5, c’est-à-dire :
    • le type de backbone (extraction des features)
    • le neck (FPN / PANet pour la fusion multi-échelle)
    • le head (détection : bounding boxes, classes, scores)
    • les paramètres de profondeur et de largeur du réseau

c. Déploiement du modèle

Une fois l’entraînement du modèle YOLOv5-PyTorch terminé, le modèle obtenu doit être chargé dans Dofbot afin de permettre la détection des déchets: b2ms/yolov5-pytorch/model4.pt

Il est alors possible de lancer la détection à l’aide de detect.py en spécifiant différents paramètres, notamment : --source, --weights, --output, etc.

Afin de tester et valider le modèle sur le Dofbot, deux approches complémentaires ont été adoptées.

✦ Détection sur images statiques

Dans un premier temps, le modèle a été testé sur un ensemble d’images statiques. Cette étape permet de :

  • vérifier le bon chargement du modèle,
  • valider la compatibilité entre le modèle et le Dofbot,
  • analyser visuellement la précision des détections (classes, bounding boxes, probabilités),
  • corriger d’éventuelles erreurs avant le passage au temps réel.

Les images sont traitées une par une, et les résultats de détection sont affichés directement sur les images, facilitant ainsi l’évaluation qualitative du modèle.

✦ Détection en temps réel avec la caméra

Une fois les tests sur images validés, le modèle est intégré à un flux vidéo issu de la caméra du Dofbot. Cette seconde approche permet de :

  • évaluer les performances du modèle en conditions réelles,
  • mesurer le temps de traitement par image,
  • tester la stabilité de la détection sur plusieurs frames consécutives,

Cette étape constitue une transition essentielle entre la validation du modèle et son exploitation opérationnelle dans le système de tri automatique.

Lors du lancement de la détection, il est possible qu’une erreur apparaisse. Celle-ci est généralement due au fait que la caméra est déjà utilisée par un autre processus, empêchant ainsi son accès par le script de détection du Dofbot.
Dans ce cas, il suffit d’arrêter le processus en cours qui utilise la caméra afin de libérer la ressource. La procédure à suivre est illustrée dans la vidéo suivante, qui montre comment identifier et interrompre le processus concerné.

Ces commandes ont été intégrées dans un script de lancement .sh, permettant d’automatiser l’initialisation de l’environnement et le démarrage de la détection. Cette approche facilite l’utilisation du système, limite les erreurs de manipulation et garantit une exécution cohérente à chaque lancement du Dofbot.

#!/bin/bash

✦ Script d’identification des déchets

Pour permettre au Dofbot d’identifier les déchets détectés par le modèle pré-entrainé, un script Python dédié a été créé : garbage_identify.py.

    Ce script a pour rôle principal de :
  • détecter les types de déchets dans l’image,
  • renvoyer les coordonnées de l’objet dans le repère de l’image.
garbage_identify.py 

#!/usr/bin/env python3
# coding: utf-8

import time
import torch
import cv2 as cv
import numpy as np
from numpy import random

from utils.torch_utils import select_device
from models.experimental import attempt_load
from utils.general import (
    non_max_suppression,
    scale_coords,
    xyxy2xywh,
    plot_one_box
)

model_path = '/home/jetson/b2ms/yolov5-pytorch/model4.pt'

# Initialize device
device = select_device()

# Load YOLO model
model = attempt_load(model_path, map_location=device)

# Get names and colors
names = model.module.names if hasattr(model, 'module') else model.names
# Get the color value randomly
colors = [[random.randint(0, 255) for _ in range(3)] for _ in range(len(names))]


class garbage_identify:
    """
    YOLO-based garbage identification class
    ROLE:
    - Detect garbage type
    - Return {class_name: (x, y)} in image coordinates
    - DOES NOT control the robot arm
    """

    def __init__(self):
        self.frame = None
        self.garbage_index = 0  # used to delay detection at startup
        self.model_ready = False
        
    def warmup(self):
        """
        Warmup YOLO model at startup
        """
        dummy = np.zeros((480, 640, 3), dtype=np.uint8)
        for _ in range(5):
            self.garbage_run(dummy)

        self.model_ready = True

    def garbage_run(self, image):
        """
        Run garbage detection
        :param image: Original camera frame
        :return: frame (with drawings), msg {name: (x, y)}
        """
        self.frame = cv.resize(image, (640, 480))
        msg = {}

        if not self.model_ready:
            cv.putText(
                self.frame,
                "MODEL LOADING...",
                (150, 50),
                cv.FONT_HERSHEY_SIMPLEX,
                1,
                (0, 0, 255),
                2
            )
            return self.frame, msg

        try:
            msg = self.get_pos()
        except Exception as e:
            print("YOLO detection error:", e)

        return self.frame, msg

    def get_pos(self):
        """
        Perform YOLO inference and extract object position
        :return: dict {class_name: (x, y)}
        """
        msg = {}

        # Copy frame to avoid modifying original
        img = self.frame.copy()

        # Preprocess
        img = np.transpose(img, (2, 0, 1))  # HWC → CHW
        img = torch.from_numpy(img).to(device)
        img = img.float() / 255.0

        if img.ndimension() == 3:
            img = img.unsqueeze(0)

        # Inference
        pred = model(img)[0]

        # Apply Non-Max Suppression
        pred = non_max_suppression(pred, 0.4, 0.5)

        gn = torch.tensor(self.frame.shape)[[1, 0, 1, 0]]

        if pred != [None]:
            for det in pred:
                if det is None:
                    continue

                # Rescale boxes
                det[:, :4] = scale_coords(
                    img.shape[2:], det[:, :4], self.frame.shape
                ).round()

                for *xyxy, conf, cls in reversed(det):
                    class_id = int(cls)
                    name = names[class_id]

                    # Bounding box center
                    xywh = (
                        xyxy2xywh(torch.tensor(xyxy).view(1, 4)) / gn
                    ).view(-1).tolist()

                    cx = int(xywh[0] * 640)
                    cy = int(xywh[1] * 480)

                    # Draw center point
                    cv.circle(self.frame, (cx, cy), 5, (0, 0, 255), -1)

                    # Draw bounding box
                    label = f"{name} {conf:.2f}"
                    plot_one_box(
                        xyxy,
                        self.frame,
                        label=label,
                        color=colors[class_id],
                        line_thickness=2
                    )

                    # Convert pixel coordinates to normalized robot coordinates
                    x = round((cx - 320) / 4000, 5)
                    y = round(((480 - cy) / 3000) * 0.8 + 0.19, 5)

                    msg[name] = (x, y)

        return msg

2. Contrôle et validation des mouvements

Avant de contrôler le bras robotisé, il est indispensable de positionner correctement le Dofbot dans son environnement de travail. Ceci permet d’anticiper les contraintes mécaniques, d’éviter les collisions potentielles et de s’assurer que les mouvements du bras seront effectifs et reproductibles dans les conditions réelles d’utilisation.

Une fois l’environnement défini, les mouvements du bras Dofbot sont contrôlés et testés indépendamment de la détection. Chaque mouvement souhaité est programmé puis validé, notamment :

  • La position initiale : permet au bras de se placer correctement pour détecter les objets sur le convoyeur.
  • La prise des objets : inclut le déplacement vers l’objet, l’ouverture/fermeture de la pince et la sécurisation de la préhension.
  • Le dépôt selon sa catégorie : transport de l’objet vers la zone de tri correspondante et retour à la position initiale.

Cette séquence permet de tester et vérifier la précision, la stabilité et la fiabilité de chaque mouvement avant leur intégration dans l’algorithme de tri automatique, garantissant ainsi un fonctionnement sûr et efficace du Dofbot.

Remarque:
Les servomoteurs du bras Dofbot peuvent être contrôlés individuellement ou simultanément. Cependant, étant donné que les servomoteurs 3 et 6 sont interdépendants (le servomoteur 6 influence directement le mouvement du 3), il est préférable de séparer les commandes autant que possible. D'ailleurs, la commande sur le 3 seul ne fonctionne pas.
Le contrôle individuel est effectué par la commande: Arm.Arm_serial_servo_write(id, angle, mvt_time)

    Cette approche permet de :
  • limiter les conflits entre mouvements,
  • garantir une précision accrue lors de la préhension et du dépôt,
  • faciliter la programmation et le débogage des séquences de mouvement.

En pratique, la séquence de tri est donc conçue pour commander chaque servomoteur individuellement, tout en synchronisant les mouvements essentiels pour assurer un fonctionnement fluide et sûr.

Après validation des mouvements à l’aide de Jupyter Notebook, un script Python dédié est créé afin d’être utilisé directement dans le processus de tri automatique. Ce script regroupe l’ensemble des séquences de mouvement validées et permet une intégration simple et efficace avec l’algorithme de détection et de décision du système de tri.

garbage_grab.py 

#!/usr/bin/env python
# coding: utf-8
import Arm_Lib
from time import sleep

class garbage_grap_move:
    def __init__(self):
        # set move status
        self.move_status = True
        self.arm = Arm_Lib.Arm_Device()
        # Clamping jaw tightening angle
        self.grap_joint = 134
        self.mvt_time = 550
        self.turn_dangereux = 140
        self.turn_menagers = 20
        self.arm.Arm_RGB_set(0, 0, 0) #RGB off
        sleep(.5)


    def mvt_pos_initial(self):
        '''
        initial position to detect object
        '''
        self.arm.Arm_RGB_set(0, 0, 0) #RGB off
        sleep(.1)
        
        self.arm.Arm_serial_servo_write(6, 120, self.mvt_time + 400)  # servo 3, 6
        sleep(0.2)
        self.arm.Arm_serial_servo_write(1, 90, self.mvt_time + 400)   # servo 1
        sleep(0.2)
        self.arm.Arm_serial_servo_write(2, 90, self.mvt_time + 400)   # servo 2
        sleep(0.2)
        self.arm.Arm_serial_servo_write(4, 50, self.mvt_time + 400)   # servo 4
        sleep(0.2)
        self.arm.Arm_serial_servo_write(5, 90, self.mvt_time + 400)   # servo 5
        sleep(0.2)
        self.arm.Arm_serial_servo_write(6, 175, self.mvt_time + 400)  # servo 3, 6
        sleep(1)
        
    def mvt_pos_object(self):
        '''
        position on object
        '''
        self.arm.Arm_serial_servo_write(5, 270, self.mvt_time)   # servo 5
        sleep(0.2)
        self.arm.Arm_serial_servo_write(4, 70, self.mvt_time)   # servo 4
        sleep(0.2)
        self.arm.Arm_serial_servo_write(6, 100, self.mvt_time)   # servo 6
        sleep(0.5)
        self.arm.Arm_serial_servo_write(2, 16, 3*self.mvt_time)   # servo 2
        sleep(2)
    
    def mvt_take_object(self):
        '''
        position on object
        '''
        self.arm.Arm_serial_servo_write(2, 22, 200)   # servo 2
        sleep(0.2)

        self.arm.Arm_serial_servo_write(6, 110, self.mvt_time + 200)   # servo 6
        sleep(0.4)

        self.arm.Arm_serial_servo_write(2, 28, self.mvt_time + 200)   # servo 2
        sleep(0.2)

        self.arm.Arm_serial_servo_write(6, 125, self.mvt_time + 200)   # servo 6
        sleep(0.4)

        self.arm.Arm_serial_servo_write(2, 35, self.mvt_time + 200)   # servo 2
        sleep(0.8)

        self.arm.Arm_serial_servo_write(6, 134, self.mvt_time + 200)   # servo 6
        sleep(0.3)

        self.arm.Arm_serial_servo_write(2, 44, self.mvt_time + 200)   # servo 2
        sleep(1)
        
    def mvt_up(self):
        '''
        move up object
        '''
        self.arm.Arm_serial_servo_write(2, 70, self.mvt_time)   # servo 2
        sleep(0.5)
    
    def grab_object(self):
        self.mvt_pos_object()
        self.mvt_take_object()
        self.mvt_up()

        
    def mvt_dangereux(self):
        '''
        recycle hazardous waste
        '''
        self.arm.Arm_RGB_set(50, 0, 0) #RGB red
        sleep(.5)
        
        self.grab_object()
        
        self.arm.Arm_serial_servo_write(1, self.turn_dangereux, self.mvt_time)  # servo 4
        sleep(0.2)
        self.arm.Arm_serial_servo_write(4, 30, self.mvt_time)   # servo 4
        sleep(0.4)
        self.arm.Arm_serial_servo_write(6, 120, self.mvt_time)   # servo 6
        sleep(0.8)
        
        
    def mvt_menagers(self):
        '''
        recycle kitchen waste
        '''
        self.arm.Arm_RGB_set(0, 0, 50) #RGB blue
        sleep(.5)
        
        self.grab_object()
        
        self.arm.Arm_serial_servo_write(1, self.turn_menagers, self.mvt_time)  # servo 4
        sleep(0.2)
        self.arm.Arm_serial_servo_write(4, 30, self.mvt_time)   # servo 4
        sleep(0.4)
        self.arm.Arm_serial_servo_write(6, 120, self.mvt_time)   # servo 6
        sleep(0.8)
        
    def mvt_recyclables(self):
        '''
        recycle recyclable waste
        '''
        self.arm.Arm_RGB_set(0, 50, 0) #RGB green
        sleep(.5)
        
        self.grab_object()
        
        self.arm.Arm_serial_servo_write(2, 175, self.mvt_time*2)    # servo 2
        sleep(0.8)
        self.arm.Arm_serial_servo_write(4, 120, self.mvt_time)  # servo 4
        sleep(0.6)
        self.arm.Arm_serial_servo_write(6, 120, self.mvt_time)   # servo 6
        sleep(0.8)

    def mvt_others(self):
        '''
        recycle others waste: not identified
        '''
        self.arm.Arm_RGB_set(255, 165, 0) #RGB orange
        sleep(.5)
        
        self.grab_object()
        
        self.arm.Arm_serial_servo_write(2, 175, self.mvt_time)    # servo 2
        sleep(1)
        self.arm.Arm_serial_servo_write(4, 120, self.mvt_time)  # servo 4
        sleep(2)
        self.arm.Arm_serial_servo_write(6, 120, self.mvt_time)   # servo 6
        sleep(2)
        
    def arm_run(self, name):
        '''
        Manipulator movement function
        :param name: Identified spam names
        '''
        # Hazardous waste - red
        if name == "dangereux" and self.move_status == True:
            # It is set here. You can only run down after this operation
            self.move_status = False
            # print("Hazardous waste")
            self.mvt_dangereux()
            self.move_status = True
            
        # Kitchen waste - blue
        if name == "menagers" and self.move_status == True:
            self.move_status = False
            # print("Kitchen waste")
            self.mvt_menagers()
            self.move_status = True
            
        # Recyclable waste - green
        if name == "recyclables" and self.move_status == True:
            self.move_status = False
            # print("Recyclable waste")
            self.mvt_recyclables()
            self.move_status = True
        
        self.mvt_pos_initial()

3. Algorithme de tri

Le script garbage_sorting.py centralise l’ensemble des opérations nécessaires au tri automatique des déchets par le Dofbot. Il combine la détection via YOLOv5 et les mouvements du bras robotisé, tout en gérant la stabilité et la répétabilité des opérations.

garbage_sorting.py 

#!/usr/bin/env python
# coding: utf-8

import threading
import cv2 as cv
from time import sleep
from garbage_identify import garbage_identify
from garbage_grap import garbage_grap_move


class garbage_sorting:
    def __init__(self):
        self.name_tmp = None
        self.num = 0
        self.status = 'waiting'
        self.nbr_detection = 2

        self.garbage_identify = garbage_identify()
        self.garbage_identify.warmup()

        self.arm = garbage_grap_move()
        self.arm.mvt_pos_initial()

    def single_garbage_run(self, image, piece_present=False):
        self.frame = cv.resize(image, (640, 480))

        if not self.garbage_identify.model_ready:
            cv.putText(
                self.frame,
                "MODEL LOADING...",
                (150, 50),
                cv.FONT_HERSHEY_SIMPLEX,
                1,
                (0, 0, 255),
                2
            )
            return self.frame

        if not piece_present:
            self.arm.arm.Arm_RGB_set(0, 0, 0) #RGB off
            cv.putText(
                self.frame,
                "WAITING FOR PIECE",
                (150, 50),
                cv.FONT_HERSHEY_SIMPLEX,
                1,
                (0, 0, 255),
                2
            )
            return self.frame

        if self.status == 'waiting':
            self.garbage_getName()

        return self.frame


    def garbage_getName(self):
        if self.status != "waiting":
            return

        self.frame, msg = self.garbage_identify.garbage_run(self.frame)
        self.arm.arm.Arm_RGB_set(255, 165, 0) #RGB orange

        name = "None"
        for key in msg.keys():
            name = key

        if name == self.name_tmp and name != "None":
            self.num += 1
            if self.num >= self.nbr_detection:
                self.status = 'running'
                threading.Thread(
                    target=self.sorting_garbage,
                    args=(name,),
                    daemon=True
                ).start()
                self.num = 0
        else:
            self.name_tmp = name
            self.num = 0

    def sorting_garbage(self, name):
        self.arm.arm.Arm_Buzzer_On(1)
        sleep(0.5)

        if name == "dangereux":
            self.arm.mvt_dangereux()
        elif name == "menagers":
            self.arm.mvt_menagers()
        elif name == "recyclables":
            self.arm.mvt_recyclables()
        else:
            self.arm.mvt_others()

        self.arm.mvt_pos_initial()
        self.status = 'waiting'

a. Detection d'objets

  • Après l’initialisation du système, le flux vidéo est analysé image par image.
  • Le modèle YOLO retourne un dictionnaire msg contenant les objets détectés et leurs coordonnées dans l’image.
  • La méthode garbage_getName vérifie la détection sur plusieurs frames consécutives afin d’éviter les faux positifs.

b. Validation de la détection

  • name_tmp : mémorise la dernière classe détectée.
  • num : compteur d’apparitions consécutives.
  • nbr_detection : nombre minimal de détections consécutives pour valider l’objet.

Si le même objet est détecté sur au moins nbr_detection frames, la détection est considérée valide et le tri peut commencer.
Sinon, le système continue de scanner le flux vidéo.

c. Tri et mouvement

  • Une fois la détection validée, le tri est déclenché dans un thread séparé, pour ne pas bloquer la détection en continu.
  • L’objet est trié selon sa catégorie (dangereux, menagers, recyclables, autres).
  • Le bras retourne ensuite automatiquement à la position initiale.

d. Logique de fonctionnement

    L’algorithme suit une boucle continue permettant au Dofbot de détecter et trier les déchets automatiquement :
  • Capture d’image : le convoyeur est scanné en temps réel pour détecter la présence d’objets.
  • Détection des objets : le script identifie les objets présents et enregistre leur classe.
  • Validation de la détection : un objet doit être détecté sur plusieurs frames consécutives pour être considéré comme valide.
  • Tri via le bras robotisé: l'objet est trié à la suite d'une sequence de mouvement du bras.
  • Signal et Visualisation : le buzzer du bras indique le début de l’opération, et la led RGB indique le type de déchet en cours de traitement.
  • Répétition : le système revient à l’état “attente” pour continuer la surveillance du convoyeur.

Cette logique garantit un tri fiable, précis et visuellement identifiable, tout en maintenant la détection active en continu.

Cette phase vise à évaluer le fonctionnement global du système de tri intelligent en conditions réelles, en comparant deux modes de communication entre le convoyeur et le Dofbot : communication sans fil et communication filaire via UART. a. Tests avec communication sans fil

Dans cette configuration, la synchronisation entre le convoyeur et le Dofbot repose sur une communication sans fil via TCP. Le convoyeur agit comme émetteur d’état (présence ou absence d’un déchet), tandis que le Dofbot reçoit cette information et déclenche la détection puis le tri.
Le script ci-dessous permet de tester le système complet (détection et tri) avec une communication TCP entre le convoyeur et le Dofbot.

import cv2 as cv

Lors des premiers essais basés sur une communication sans fil entre le convoyeur et le Dofbot, plusieurs limites ont été observées :

  • Une latence notable entre l’envoi du signal par le convoyeur et la réaction du Dofbot, impactant la synchronisation entre détection et action.
  • Les déchets ont tendance à ne pas être correctement centrés sur la zone de préhension, ce qui entraîne des échecs de saisie par la pince.
  • Le temps global du cycle de tri est relativement élevé, dû à la durée du convoyage et la lenteur volontaire des mouvements du bras afin de préserver la stabilité.

Ces contraintes montrent que, bien que fonctionnelle, la communication sans fil n’est pas optimale pour un tri précis et répétable dans cette configuration.
Ce mode reste néanmoins intéressant pour une architecture distribuée ou lorsque le câblage n’est pas possible.

5. Suivi et gestion des historiques

À la suite des différentes phases de tests et de démonstration, il est apparu nécessaire d’ajouter des fonctionnalités permettant de suivre et enregistrer l’historique des déchets triés. Ces ajouts visent à améliorer la traçabilité, la visualisation des performances du système, ainsi que l’exploitation des données sur une période donnée.

L’intégration de l’historique permet de :

  • Conserver une trace des déchets détectés et triés.
  • Analyser la répartition des catégories de déchets.
  • Faciliter le débogage et l’amélioration continue du tri.
  • Disposer de données exploitables pour des statistiques ou une interface future.

a. Principe de fonctionnement

À chaque opération de tri validée :

  • Le type de déchet identifié est enregistré.
  • La date et l’heure de la détection sont associées à l’événement.
  • L’information est stockée localement dans une structure dédiée.
  • Les données peuvent être consultées, analysées et utilisées ultérieurement.

Ce mécanisme est indépendant du mode de communication utilisé entre le Dofbot et le convoyeur.

b. Structure

Un module dédié à la gestion de l’historique est intégré au système.
Il assure:

  • l’enregistrement des événements de tri,
  • la mise à jour des compteurs par catégorie,
  • la persistance des données (fichier local).

garbage_history.py 

import json
import os
from datetime import datetime


class GarbageHistory:
    GARBAGE_TYPES = ["dangereux", "menagers", "recyclables"]

    def __init__(self, file_path="garbage_history.json"):
        self.file_path = file_path
        self.data = {
            "total": 0,
            "by_type": {g: 0 for g in self.GARBAGE_TYPES},
            "history": []
        }
        self.load()

    def load(self):
        if os.path.exists(self.file_path):
            with open(self.file_path, "r") as f:
                self.data = json.load(f)

            for g in self.GARBAGE_TYPES:
                if g not in self.data["by_type"]:
                    self.data["by_type"][g] = 0
        else:
            self.save()

    def save(self):
        with open(self.file_path, "w") as f:
            json.dump(self.data, f, indent=4)

    def add(self, garbage_type):
        self.data["total"] += 1
        self.data["by_type"][garbage_type] += 1

        self.data["history"].append({
            "type": garbage_type,
            "time": datetime.now().strftime("%Y-%m-%d %H:%M:%S")
        })

        self.save()
        self.display_last(garbage_type)

    def display_last(self, garbage_type):
        print(f"\nNouveau déchet trié : {garbage_type}")
        print("Répartition :")
        for g in self.GARBAGE_TYPES:
            print(f"   - {g} : {self.data['by_type'][g]} ({(self.data['by_type'][g]/self.data['total'])*100.0})%")
        print(f"Total déchets : {self.data['total']}")
        print("-" * 40)

    def display_all(self):
        print("\nHISTORIQUE COMPLET")
        for i, item in enumerate(self.data["history"], 1):
            print(f"{i}. {item['time']}: {item['type']}")

    def reset(self):
        confirm = input("Confirmer le reset ? (oui/non) : ")
        if confirm.lower() == "oui":
            self.data = {
                "total": 0,
                "by_type": {g: 0 for g in self.GARBAGE_TYPES},
                "history": []
            }
            self.save()
            print("Historique réinitialisé")
        else:
            print("Reset annulé")
