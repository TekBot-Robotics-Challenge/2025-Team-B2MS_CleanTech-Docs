Détection des déchets

Le modèle de détection, d'indentification et de classement des déchets se fera via YOLO.
YOLO(You Only Look Once) est un réseau de neurones convolutionnel (CNN) conçu pour la détection d’objets en temps réel.
Parmi les différentes versions de YOLO, YOLOv8 a été retenue comme référence dans cette documentation, car elle présente un meilleur compromis entre vitesse d’exécution et exactitude des détections. Par la suite, une comparaison entre YOLOv5 et YOLOv8 sera faite afin de déterminer la version la plus appropriée pour le Dofbot Yahboom.

2. Collecte et annotation des images

Pour pouvoir entrainer le modèle de détection, il faut un dataset: des images étiquettées. Un jeu de 100 à 200 images constitue généralement un bon point de départ pour développer un prototype de modèle, mais il faut l'enrichir le plus possible. Les images utilisées pour l'entraînement doivent présenter les objets dans différents contextes, avec des angles de vue et des éclairages variés, similaires à ceux rencontrés dans les conditions réelles d’utilisation. Il est également recommandé d'ajouter des objets non pertinents dans les images afin d'aider le modèle à différencier les éléments à détecter de ceux à ignorer.

3. Structure des dossiers

Ultralytics exige une structure de dossiers particulière pour les données d'entraînement des modèles. Le dossier racine est nommé data. À l'intérieur, il y a deux dossiers principaux :

  • train contient les images et les labels utilisés pour entraîner le modèle. L'algorithme d'entraînement ajuste le modèle en fonction des données présentes dans ces images.
  • valid contient les images et les labels qui sont utilisés périodiquement pour tester le modèle pendant l'entraînement. À la fin de chaque époque d'entraînement, le modèle est évalué sur ces images afin de calculer des métriques comme la précision, le rappel et le mAP.

En général, 80% des images d'un dataset sont placées dans le dossier train et 20% dans validation. Dans chacun de ces dossiers se trouvent un dossier images et un dossier labels, qui contiennent respectivement les fichiers d'images et les fichiers d'annotations.
La structure globale des dossiers requise pour entraîner les modèles YOLO d'Ultralytics est illustrée ci-dessous.

  • data

      • images

        • image3.jpg
        • ...
      • labels
        • image1.txt
        • image2.txt
        • image3.txt
        • ...
      • classes.txt

      • train
        • image101.jpg

        • image103.jpg
        • ...

organisation des dossiers 

# Paramètres
data_path = "./dataset"     # Chemin du dataset d'origine
train_percent = 0.8         # Pourcentage d'images pour l'entraînement (train), le reste sera utilisé pour la validation (val)


# Vérification des paramètres
if not os.path.isdir(data_path):
    print("Le dossier spécifié par data_path est introuvable. Vérifie le chemin et réessaie.")
    sys.exit(0)

if train_percent < 0.01 or train_percent > 0.99:
    print("Valeur invalide pour train_percent. Elle doit être comprise entre 0.01 et 0.99.")
    sys.exit(0)

val_percent = 1 - train_percent


# Définition des chemins
input_image_path = os.path.join(data_path, "images")  # Dossier contenant les images originales
input_label_path = os.path.join(data_path, "labels")  # Dossier contenant les labels

val_label_path = os.path.join(cwd, "data/valid/labels")


# Nettoyage et création des dossiers
for path in [train_img_path, train_label_path, val_img_path, val_label_path]:
    if os.path.exists(path):
        shutil.rmtree(path)  # Supprime les anciens fichiers pour éviter les doublons
    os.makedirs(path)  # Crée les nouveaux dossiers vides
    print(f"Dossier prêt : {path}")


# Récupération des fichiers
img_file_list = [path for path in Path(input_image_path).rglob("*") if path.is_file()]
txt_file_list = [path for path in Path(input_label_path).rglob("*") if path.is_file()]

print(f"\nNombre total d'images : {len(img_file_list)}")
print(f"Nombre total de fichiers d'annotation : {len(txt_file_list)}")


# Détermination du nombre de fichiers à déplacer
file_num = len(img_file_list)
train_num = int(file_num * train_percent)
val_num = file_num - train_num

print(f"\nImages vers train : {train_num}")
print(f"Images vers valid : {val_num}")


# Mélange aléatoire des images
random.shuffle(img_file_list)
train_files = img_file_list[:train_num]
val_files = img_file_list[train_num:]

# Train
for img_path in tqdm(train_files, desc="Copie des fichiers train"):
    img_fn = img_path.name
    base_fn = img_path.stem
    txt_fn = base_fn + ".txt"
    txt_path = os.path.join(input_label_path, txt_fn)

    # Copie de l'image
    shutil.copy(img_path, os.path.join(train_img_path, img_fn))

    # Copie du label si disponible
    if os.path.exists(txt_path):
        shutil.copy(txt_path, os.path.join(train_label_path, txt_fn))

# Validation
for img_path in tqdm(val_files, desc="Copie des fichiers validation"):
    img_fn = img_path.name
    base_fn = img_path.stem
    txt_fn = base_fn + ".txt"
    txt_path = os.path.join(input_label_path, txt_fn)

    shutil.copy(img_path, os.path.join(val_img_path, img_fn))

    if os.path.exists(txt_path):
        shutil.copy(txt_path, os.path.join(val_label_path, txt_fn))


print("\nDivision du dataset terminée avec succès !")

print(f"Validation : {len(os.listdir(val_img_path))} images")

c. Configuration de l'entraînement

Cette étape permet de définir le modèle YOLO pré-entraîné à utiliser ainsi que tous les paramètres nécessaires pour l'entraînement(nombre d'époques, taille des images, batch, learning rate, etc.).

Configuration d'entrainement 

# Chargement du modèle YOLOv8 pré-entraîné
model = YOLO('yolov8n.pt')  # nano version (plus rapide)

# Configuration de l'entraînement
epochs = 20
imgsz = 640
batch = 16

train_config = {
    'data': 'data.yaml', 
    'epochs': epochs,            
    'imgsz': imgsz,          
    'batch': batch,            
    'device': 0 if torch.cuda.is_available() else 'cpu',
    'workers': 4,           
    'project': 'runs/detect',  # Nom du projet
    'name': 'train',        # Nom de l'expérience
    'save_period': 5,       # Sauvegarde tous les 5 epochs
    'patience': 5,          # Early stopping
    'lr0': 0.01,           # Learning rate initial
    'warmup_epochs': 1,     # Epochs de warmup
    'verbose': True
}

print("Configuration d'entraînement :")
for key, value in train_config.items():
    print(f"  {key}: {value}")
Paramètre Description Impact
data Chemin vers le fichier YAML contenant les informations du dataset
epochs Nombre total d'époques pour l'entraînement(Nombre de fois que le modèle parcourt l'ensemble du dataset)
  • Trop petit: le modèle peut ne pas apprendre correctement.
  • Trop grand: risque de sur-apprentissage, et entraînement plus long.
imgsz Taille des images utilisées pour l'entraînement
  • Plus petit: rapide mais perte de détails, précision réduite
  • Plus grand: meilleure précision possible, mais plus de mémoire GPU nécessaire, entraînement plus lent.
batch Nombre d'images traitées simultanément
  • Petit batch: moins de mémoire nécessaire, mais entraînement plus bruité et instable.
  • Grand batch: convergence plus stable, mais plus de mémoire GPU nécessaire.
device Appareil utilisé : GPU (0) si disponible, sinon CPU
  • GPU: entraînement beaucoup plus rapide, recommandé si disponible.
  • CPU: très lent, déconseillé pour l’entraînement de modèles complexes.

e. Visualisation des métriques

Après l'entraînement, il est important de visualiser les métriques pour évaluer les performances du modèle, comme la précision, le rappel, etc. Cela permet de vérifier si le modèle apprend correctement et d'identifier d'éventuels problèmes.

Visualisation des métriques 

results_path = Path(f"{train_config["project"]}/{train_config["name"]}")
csv_path = results_path / 'results.csv'

if csv_path.exists():
    # Lecture des métriques
    df = pd.read_csv(csv_path)

    else:

                  'time' and c != 'lr/pg0' and c != 'lr/pg1' and c != 'lr/pg2']

        n = len(metrics)
        cols = 5
        rows = math.ceil(n / cols)

        #Affichage de l'image sur jupyter notebook
        %matplotlib inline  
        fig, axes = plt.subplots(rows, cols, figsize=(5 * cols, 4 * rows))
        axes = axes.flatten() if n > 1 else [axes]

        # Boucle sur chaque métrique
        for i, col in enumerate(metrics):
            axes[i].plot(df['epoch'], df[col], '-o', linewidth=2)
            axes[i].set_title(col, fontsize=10)
            axes[i].set_xlabel('epoch')

        print("\n=== MÉTRIQUES FINALES ===")

            if pd.api.types.is_numeric_dtype(type(val)):

Métrique Description
train/box_loss Erreur sur la localisation des boîtes englobantes pendant l’entraînement
train/cls_loss Erreur sur la classification des objets dans les boîtes pendant l’entraînement
train/dfl_loss Erreur de distribution des distances pour affiner les boîtes (Distance Focal Loss)
val/box_loss Erreur sur la localisation des boîtes sur le dataset de validation
val/cls_loss Erreur sur la classification des objets sur le dataset de validation
val/dfl_loss Erreur de distribution des distances sur le dataset de validation
metrics/precision(B) Précision du modèle: proportion de détections correctes sur toutes les détections(batch)
metrics/recall(B) Rappel: proportion d'objets réellement présents correctement détectés(batch)
metrics/mAP50(B) Mean Average Precision à IoU 0.5: qualité globale de détection(batch)
metrics/mAP50-95(B) Mean Average Precision moyenne sur IoU de 0.5 à 0.95(batch)

def predict_and_visualize(model, image_path, conf_threshold=0.5):
    """
    Faire une prédiction et visualiser les résultats
    """
    # Prédiction
    results = model(image_path, conf=conf_threshold)

    # Visualisation
    annotated_image = results[0].plot()

    # Conversion BGR -> RGB pour matplotlib
    annotated_image = cv2.cvtColor(annotated_image, cv2.COLOR_BGR2RGB)

    # Affichage
    #Affichage de l'image sur jupyter notebook
    %matplotlib inline     
    print("Affichage de l'image")
    plt.figure(figsize=(12, 8))
    plt.imshow(annotated_image)
    plt.axis('off')
    plt.title(f'Détections (Confiance > {conf_threshold})')
    plt.show()

    # Informations sur les détections
    boxes = results[0].boxes
    if boxes is not None:
        print(f"{len(boxes)} objets détectés :")
        for i, box in enumerate(boxes):
            class_id = int(box.cls[0])
            confidence = float(box.conf[0])
            class_name = model.names[class_id]
            print(f"  {i+1}. {class_name} (confiance: {confidence:.2f})")
    else:
        print("Aucun objet détecté")

5. Scripts python

Les codes sont regroupés sous forme de scripts Python pour simplifier leur utilisation. Cela permet de configurer facilement les paramètres selon les besoins, sans avoir à modifier directement les lignes de code, rendant ainsi le système plus flexible et accessible. Cette modularité est possible grâce à argparse, qui permet de passer des options directement en ligne de commande, avec des valeurs par défaut et une aide automatique pour l’utilisateur.

a. Environnement de travail

Il faut toujours commencer par activer/configurer son environnement virtuel et créer un dossier de travail: 


conda activate b2ms-env
pip install -qU ultralytics opencv-python matplotlib pillow torch
pip install -qU --upgrade torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124  #pour pc avec gpu
mkdir detection
cd detection

Il faut ensuite copier le dossier scripts dans le repertoire de travail.

b. Préparation du Dataset


python scripts/split_dataset.py --dataset_path="dataset" --train_per=0.8
  • --dataset_path : Chemin du dossier contenant les données (dataset/).
  • --train_per : Pourcentage du dataset utilisé pour l'entraînement (par défaut 0.8: 80%).

c. Entraînement du modèle


python scripts/training_model.py --model=yolov8n.pt --epochs=20 --imgsz=640 --batch=16
  • --model : Modèle YOLO de départ (yolov8n.pt, yolov8s.pt, etc.).
  • --epochs : Nombre d'itérations d'apprentissage (défaut 20).
  • --imgsz : Taille des images d'entrée (défaut 640).
  • --batch : Taille du batch (défaut 16).

Les résultats d'entraînement (modele entrainé, logs, métriques) sont enregistrés dans : runs/detect/train/

d. Visualisation des métriques


python scripts/metric_visualization.py --result_path=runs/detect/train/results.csv
  • --result_path : Chemin du fichier results.csv généré par l’entraînement (par défaut runs/detect/train/results.csv).

e. Evaluation sur données de validation


python scripts/evaluate_val.py --model_path=runs/detect/train/weights/best.pt --imgsz=640
  • --model_path : Chemin du meilleur modèle (par défaut runs/detect/train/weights/best.pt).
  • --imgsz : Taille des images à l’évaluation (défaut 640).

f. Lancement du modèle


python scripts/detection_launch.py --model=runs/detect/train/weights/best.pt --source=usb0 --confidence=0.6 --resolution=640x640
  • --model : Chemin vers le modèle entraîné.
  • --source : Source des données :
    • image.jpg: une image unique
    • data/images/: un dossier

    • "usb0": caméra USB
  • --confidence : Seuil minimal de confiance (défaut 0.5).

  • --export_to : Dossier de destination pour les fichiers exportés (défaut : models).
  • --name : Nom du modèle exporté (défaut : my_model).
  • --imgsz : Taille des images à l'export (défaut 640).

b. Comparaison sur un ordinateur

Les tests réalisés sur un ordinateur sous Windows équipé d’un GPU montrent que YOLOv8 et YOLOv5 présentent des performances globalement proches, avec quelques erreurs observées(non-détection ou classification incorrecte).
En termes de vitesse, les résultats restent élevés et relativement similaires : YOLOv8 oscille entre 20.74 et 21.58 FPS, tandis que YOLOv5 varie entre 18.47 et 20.65 FPS.
Pour la précision, YOLOv8 se montre légèrement meilleur, même si les deux modèles présentent parfois des erreurs similaires.
Dans l’ensemble, sur cette machine, les deux modèles offrent des performances comparables, avec un léger avantage pour YOLOv8 en précision et une vitesse globalement équivalente.
